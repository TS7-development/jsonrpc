void ts7::jsonrpc::examples::any_collection::storage ( )

storage example

Demonstrates how to store and call print with different types, which is working fine with the boost::any type.

Advantage of that solution is, that there is no need to adjust that Storage, if a new type is added since boost::any can store any type. In exchange the Storage needs to be adjusted for every new member function that shall be suppored, which happens way less than new types.

◆ variant_storage()

void ts7::jsonrpc::examples::any_collection::variant_storage ( )

variant_storage example

This example demonstrates how to store elements by using boost::variant.

This has worked quite fine. But compared to boost:any example storage, this has some problems. First we always need to update variant_type to support every new type (i.e. every template combination for a JSON RPC callback signature). Secondly, for every new method that shall be callable on those type, we also need to implement a new visitor. This is basically the same workload as with boost::any that requires a new callback lambda for every new supported member function.
