constexpr bool ts7\+::jsonrpc\+::playground\+::json\+\_\+call\+::always\+\_\+false = false\hspace{0.3cm}{\ttfamily [constexpr]}}



Always false. 

This bool is a evaluated at compile time and will be always false no matter which and how many types are provided to the variadic template. This is required for static\+\_\+assert. If you just pass false to it, the compiler will optimize it and directly fail, because it will be false no matter which type is provided. This bool on the other hand requires the types to be evaluated to it can\textquotesingle{}t be directly optimized away and will only lead to the static\+\_\+assert fail, if a certain type is used.

This approach is decribed in\+: \href{https://artificial-mind.net/blog/2020/10/03/always-false}{\texttt{ https\+://artificial-\/mind.\+net/blog/2020/10/03/always-\/false}}
